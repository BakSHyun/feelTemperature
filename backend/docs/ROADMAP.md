# 개발 로드맵

## 단기 (1-2주)

### 필수
- [ ] 단위 테스트 커버리지 70% 이상
- [ ] 통합 테스트 추가
- [ ] API 문서화 (Swagger)

### 개선
- [ ] 서비스 인터페이스 명확화 (마이크로서비스 전환 준비)
- [ ] API 버전 관리 도입 (`/api/v1/...`)

---

## 중기 (1-2개월)

### 필수
- [ ] Redis 캐싱 도입
- [ ] 비동기 처리 (기록 생성)
- [ ] 로그 수집 시스템 연동

### 마이크로서비스 전환 준비
- [ ] 서비스 간 직접 의존성 제거 (인터페이스로 통일)
- [ ] 이벤트 기반 통신 구조 설계
- [ ] 도메인 이벤트 정의
- [ ] API Gateway 프로토타입

---

## 장기 (3-6개월)

### 마이크로서비스 전환
1. **Question Service 분리** (가장 독립적)
   - 독립적인 도메인
   - 다른 서비스와 약한 결합
   - 우선 분리 대상

2. **Matching Service 분리**
   - 매칭 도메인 분리
   - Participant 도메인 포함

3. **Answer Service 분리**
   - 응답 도메인 분리
   - Record Service와의 통신 고려

4. **Record Service 분리**
   - 기록 도메인 분리
   - 여러 서비스와 통신 필요

5. **Temperature Calculation Service 분리**
   - Stateless 서비스
   - 계산만 수행

### 최적화
- [ ] 서비스 메시 도입 (Istio/Linkerd)
- [ ] 이벤트 소싱 고려
- [ ] CQRS 패턴 적용 (필요시)
- [ ] 분산 추적 시스템 (Zipkin/Jaeger)
